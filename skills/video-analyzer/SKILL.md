# Video Analyzer

Локальный анализ видео для продакт-менеджера. Принимает **путь к файлу** или **YouTube-ссылку**,
извлекает кадры и речь полностью локально (ffmpeg + WhisperX, без облака), и выдаёт анализ,
адаптированный под жанр видео.

## Жанры (выбирай стратегию под жанр)

| Жанр | Признаки | Фокус анализа |
|---|---|---|
| **Созвон / интервью** | локальный файл, несколько говорящих, диалог | речь: транскрипт + диаризация → продуктовое саммари |
| **Демо инструмента / конкурента** | чаще YouTube, один голос/закадр, показ экрана | **визуал**: кадры → разбор продукта (это не диалог) |
| **Гибрид** | созвон, где клиент шарит экран | связать «что говорит» с «что показывает» |

## Workflow

### 0. Проверка зависимостей (один раз)
```bash
command -v ffmpeg >/dev/null && command -v yt-dlp >/dev/null && [ -d "$SKILL_DIR/.venv" ] \
  && echo OK || echo "RUN: bash $SKILL_DIR/scripts/setup.sh"
```
Если не OK — предложи запустить `bash $SKILL_DIR/scripts/setup.sh` (поставит ffmpeg/yt-dlp и whisperx,
спросит токен HuggingFace для диаризации).

### 1. Рабочая папка (временная, безопасная)
```bash
WORK="$(mktemp -d)" && chmod 700 "$WORK" && echo "$WORK"
```

### 2. Получить видео (URL или локальный файл)
```bash
VIDEO="$(bash "$SKILL_DIR/scripts/ingest.sh" "<URL-или-путь>" "$WORK" | tail -1)"
```

### 3. Извлечь кадры (по смене сцены) и аудио
```bash
bash "$SKILL_DIR/scripts/extract.sh" "$VIDEO" "$WORK"
# → $WORK/audio.wav, $WORK/frames/frame_XXXX.jpg, $WORK/frames/timestamps.txt
```
Кадры берутся по **детекции смены сцены**: на «говорящих головах» их мало, на демо/шаринге экрана —
кадр на каждое переключение UI. `timestamps.txt` (поле `pts_time`) даёт таймкод каждого кадра.

### 4. Определить жанр
Эвристика → озвучь предположение пользователю, дай поправить:
- источник YouTube + мало речи/один голос → **демо**;
- локальная запись, много кадров-переключений + речь → **гибрид**;
- локальная запись, мало кадров, диалог → **созвон**.

### 5. Транскрибировать (для созвона/гибрида; для чистого демо — опционально)
```bash
# созвон/гибрид с метками говорящих:
bash "$SKILL_DIR/scripts/transcribe.sh" "$WORK/audio.wav" "$WORK" diarize ru
# демо без диалога (или если речь — просто закадровый комментарий):
bash "$SKILL_DIR/scripts/transcribe.sh" "$WORK/audio.wav" "$WORK" plain ru
```
Транскрипт с таймкодами/метками: `$WORK/audio.srt`, `$WORK/audio.txt`, `$WORK/audio.json`.
Метки `SPEAKER_00/01...` сопоставь с ролями (клиент/интервьюер/PM) по контексту разговора.

### 6. Прочитать ключевые кадры
Используй встроенный инструмент `Read` на файлах `$WORK/frames/frame_XXXX.jpg` (он читает JPG/PNG как
изображения — отдельный vision API не нужен). Для демо смотри все кадры переключений UI; для созвона —
кадры из сегментов шаринга экрана. Сопоставляй кадр с моментом речи по таймкодам.

### 7. Синтез — вывод по жанру (см. ниже)
Сохрани полный транскрипт рядом с источником: `<имя-видео>.transcript.md`
(таймкоды + метки говорящих). Затем выдай саммари.

### 8. Очистка
```bash
rm -rf "$WORK"
```
(Не удаляй сохранённый файл транскрипта.)

## Форматы вывода

### Саммари — созвон / интервью
Контекст пользователя: PM, ведёт Discovery и анализирует работу бизнес-аналитиков и других продактов.
- **Участники и роли** (по диаризации + контексту)
- **Темы** разговора
- **Боли / потребности / задачи клиента** — с цитатами и таймкодами
- **Ключевые решения и договорённости**
- **Action items** (кто / что / когда, если прозвучало)
- **Открытые вопросы и упущенные зацепки**
- **Оценка качества интервью**: открытые vs наводящие вопросы, давали ли клиенту говорить,
  что стоило доспросить (полезно для разбора работы бизнес-аналитика)

### Саммари — демо инструмента / конкурента (визуальный анализ)
- **Что за продукт и для кого**
- **Ключевые экраны / фичи** — со ссылками на кадры и таймкоды
- **UX-флоу и паттерны**; что выделяет продукт
- **Выводы для нас**: чему поучиться, чего избегать

### Гибрид
Саммари созвона + врезка «**Что показали на экране**» по сегментам шаринга (кадр + что говорилось).

## Заметки
- Всё считается **локально**; видео/аудио никуда не отправляются. Токен HF нужен только для
  разовой загрузки модели диаризации.
- Транскрипция идёт через faster-whisper с пословными таймкодами, диаризация — через pyannote
  напрямую (см. `scripts/diarize_transcribe.py`); медленное wav2vec2-выравнивание whisperx НЕ
  используется. Спикеры назначаются по перекрытию слов с речевыми интервалами pyannote.
- Модели скачиваются автоматически при первом запуске (кэш `~/.cache/huggingface`), поэтому
  первый прогон дольше. Модель Whisper по умолчанию `small` — на этой машине (Apple Silicon,
  ctranslate2 int8) только она работает с приемлемой скоростью (~0.17× реального времени) и при
  этом чисто распознаёт русский. Модели крупнее патологически медленные даже при свободной RAM:
  `medium` ≈ 32× реального времени, `large-v3-turbo` ≈ 4.5×. Менять смысла нет, пока не сменится
  окружение (GPU/оптимизированный билд ctranslate2). Переопределение: `WHISPER_MODEL=...`.
- YouTube требует куки браузера — `ingest.sh` по умолчанию берёт их из Chrome
  (`YTDLP_COOKIES_BROWSER=chrome|safari|firefox|none` для смены/отключения).
- Пороги кадров настраиваются: `SCENE_THRESH` (0.3), `MAX_FRAMES` (80) перед вызовом `extract.sh`.
